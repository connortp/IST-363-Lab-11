Controller and Hiding Sections

Inside the main controller, we want to create a few properties on the controller that will be needed for the website. In Knockout, properties were able to be added directly on the viewmodel. In Angular, properties are added to Scope which serves as an intermediary and the "glue" between the controller and the view

  • The Scope object $scope needs to be injected into the main controller
  • Onto the scope object, create a property called displayWeather and set it to false. This boolean will be used to control which sections should be visible or not depending on the state of the application. When users first visit the site, they should only be shown the search section. Once they search for a zip code, the search section should disappear and the current and forecase weather sections should appear instead
  • Add a current property and a forecast property with no assigned value onto scope as well. These will store information received back from the weather API once a user performs a location search
  • On index.html, use the ng-if directive and the displayWeather property to render the correct sections initially as described above
